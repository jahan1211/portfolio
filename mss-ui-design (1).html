Section 05
Accessibility & Inclusion Design

All design decisions account for rural users with low literacy, limited dexterity, outdoor screen use, and varying device quality. These principles are core requirements — not afterthoughts.

🎙️
Voice-First Design
Every screen has a voice input button. The AI reads responses aloud in Urdu or Punjabi. No typing required for critical functions. TTS playback for all instructional content.
🖼️
Icon-Based Navigation
Key actions use culturally understood icons alongside text labels. Low-literacy users can navigate entirely through visual symbols and color coding without reading.
☀️
High-Contrast Outdoor
All text meets WCAG AA (4.5:1) contrast ratios. The deep plum/cream palette remains legible in bright sunlight. Font sizes minimum 11px on mobile, 14px for critical info.
👆
Large Touch Targets
All interactive elements are ≥ 44×44px. CTA buttons span full screen width. Adequate spacing prevents accidental taps. Critical actions like SOS have confirmation dialogs.
📡
Offline-First Architecture
Core content is cached on first load. All guidance, educational material, and forms work without internet. Sync queues auto-upload when connection resumes.
🌐
RTL Language Support
Interface fully mirrors for Urdu and Punjabi (right-to-left). Navigation, forms, and text all flip direction. Font rendering supports Nastaliq script for Urdu.
Section 06
Key Design Decisions

Every major design choice is annotated with its requirement source and rationale for implementation reference.

1
Warm Plum + Rose Color System
The palette avoids clinical whites and sterile blues common in health apps. Deep plum communicates trust and privacy; rose signals femininity and care without being condescending. High contrast for outdoor readability.
→ Req 1.3: Culturally sensitive design for rural women
2
3-Language Selector on First Screen
Language selection appears before any content, not buried in settings. Default auto-detects device locale. Selection persists across sessions. Critical for inclusivity in multilingual rural Pakistan.
→ Req 1.5 Obj.1: Multilingual SMS + offline guidance
3
SMS Fallback Prominently Displayed
The SMS option "No smartphone? Dial 8500" appears on the splash screen — not hidden. Gold color distinguishes it as an alternative path, not an inferior option. Ensures no user is excluded.
→ Req 3.3.1: SMS/Missed-call for low-tech users
4
Literacy Level Indicator in AI Chat
A 5-dot "literacy bar" in the chat header shows the AI's current adaptation level. Users can manually adjust. The AI uses simpler vocabulary, shorter sentences, and more icons at lower levels.
→ Req 3.3.3: AI literacy adaptation module
5
Offline Badge in Field Worker App
An amber "OFFLINE MODE" badge is permanently visible when disconnected. Pending sync count is shown prominently. Workers always know their data is safe even without internet.
→ Req 3.3.2: Offline field-worker app with sync
6
SOS Button: Full-Width, High-Contrast
The emergency SOS action spans the full grid width on the home screen and uses the most saturated red-rose tone in the palette. It is intentionally impossible to miss. Confirmation dialog prevents accidental triggers.
→ Req 3.3.5: Emergency handling + SOS alerts
7
Education Cards: Tag + Duration + Mode
Each education card shows a category tag, estimated time, and available format (Voice / Icons / Video). Users who can't read can identify suitable content by format icon alone.
→ Req 3.3.4: Education module with voice/icon-based content
8
NGO Dashboard: Stats + Trend Indicators
Monthly trend arrows (↑18%) give NGO admins instant context without needing to dig into charts. Color-coded stat cards (plum=neutral, rose=alerts, sage=success) enable at-a-glance monitoring.
→ Req 3.3.8: Admin/NGO dashboard analytics
